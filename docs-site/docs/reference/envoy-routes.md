# Envoy Route CRDs

AIPlex uses three Envoy route types, one per plane. These are generated automatically by the deploy engine — you rarely create them manually.

## MCPRoute (MCPlex)

Routes MCP protocol traffic (JSON-RPC + SSE) to MCP server pods.

```yaml
apiVersion: aigateway.envoyproxy.io/v1alpha1
kind: MCPRoute
metadata:
  name: mcp-{instance-id}
  namespace: mcplex
spec:
  parentRefs:
    - name: aiplex-gateway
  path: "/mcp/{instance-id}"
  backendRefs:
    - name: {instance-id}
      path: "/mcp"
  securityPolicy:
    oauth:
      issuer: "https://aiplex.example.com/auth/realms/aiplex"
```

Features:
- SSE support for streaming tool responses
- MCP session affinity
- Path rewriting (`/mcp/{id}` → `/mcp` on the backend)

## HTTPRoute (A2APlex)

Standard Kubernetes Gateway API HTTPRoute for A2A agent traffic.

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: a2a-{instance-id}
  namespace: a2aplex
spec:
  parentRefs:
    - name: aiplex-gateway
  rules:
    - matches:
        - path:
            type: PathPrefix
            value: /a2a/{instance-id}
      backendRefs:
        - name: {instance-id}
          port: 8080
```

## LLMRoute (LLMPlex)

Envoy AI Gateway's native LLM routing with provider-specific backends.

```yaml
apiVersion: aigateway.envoyproxy.io/v1alpha1
kind: LLMRoute
metadata:
  name: llm-{route-name}
  namespace: aiplex-system
spec:
  parentRefs:
    - name: aiplex-gateway
  rules:
    - backendRefs:
        - name: {provider}-backend
          weight: 80
        - name: {provider2}-backend
          weight: 20
      fallback:
        - name: {fallback}-backend
```

### AIServiceBackend

Each LLM provider needs a backend definition:

```yaml
apiVersion: aigateway.envoyproxy.io/v1alpha1
kind: AIServiceBackend
metadata:
  name: gemini-backend
spec:
  provider: google
  apiKey:
    secretRef:
      name: gemini-api-key
```

## Shared Policies

### SecurityPolicy (ext_authz)

Applied to all routes:

```yaml
apiVersion: gateway.envoyproxy.io/v1alpha1
kind: SecurityPolicy
metadata:
  name: aiplex-authz
spec:
  extAuth:
    grpc:
      backendRef:
        name: opa-ext-authz
        port: 9191
    withRequestBody:
      maxRequestBytes: 65536
```

### BackendTrafficPolicy (Rate Limiting)

```yaml
apiVersion: gateway.envoyproxy.io/v1alpha1
kind: BackendTrafficPolicy
metadata:
  name: global-rate-limit
spec:
  rateLimit:
    type: Global
    global:
      rules:
        - clientSelectors:
            - headers:
                - name: x-jwt-sub
                  type: Distinct
          limit:
            requests: 200
            unit: Minute
```

## Auto-Generation

All route CRDs are generated by the deploy engine. You control them through:

- `aiplex deploy` / `aiplex apply` — creates routes on deploy
- `aiplex rm` — deletes routes on undeploy
- `aiplex.yaml` — declarative route configuration

Direct CRD manipulation is an escape hatch for platform engineers.
